# 趋势报告与研究空白分析规则说明

> **版本**: v1.0  
> **日期**: 2026-02-28  
> **阶段**: Phase 1 - 趋势与空白分析机制标准化  
> **性质**: 知识库从"结构化判断"到"选题支持系统"的升级规范

---

## 一、总体目标

### 1.1 为什么需要趋势报告与研究空白分析

| 问题 | 单篇/横向比较的局限 | 趋势与空白分析的解决 |
|------|-------------------|---------------------|
| 只见树木不见森林 | 聚焦具体论文或局部比较 | 提炼方向级别的演化逻辑 |
| 难以把握研究格局 | 缺乏对主/次/边缘路线的认知 | 识别主路线、次级路线和边缘路线 |
| 难以判断时机 | 不知道哪些方向在升温/饱和 | 判断方向生命周期状态 |
| 难以发现切入点 | 被动阅读，主动发现不足 | 识别空白问题、薄弱环节 |
| 难以支持选题 | 知识库停留在归档和判断 | 升级为选题支持系统 |

### 1.2 核心作用

1. **提炼演化逻辑**: 从大量论文中提炼方向级别的演化规律
2. **识别路线格局**: 区分主路线、次级路线和边缘路线
3. **判断温度趋势**: 识别升温方向 vs 饱和方向
4. **发现研究空白**: 发现文献体系中的空白问题和薄弱环节
5. **支持选题决策**: 为论文写作、研究规划、方法设计提供依据
6. **系统化依据**: 将知识库升级为"选题支持系统"

### 1.3 黄金标准

> **一份合格的趋势报告，应该让我能够：**
> - 画出该方向的研究路线图（含主/次/边缘路线）
> - 判断该方向当前处于生命周期的哪个阶段
> - 指出哪些子方向值得投入，哪些应该回避
> - 识别出真正有价值的研究空白（而非冷门但无价值的问题）
> - 形成具体的选题建议或研究规划

## 二、触发机制

### 2.1 数量触发阈值

| 报告类型 | 触发条件 | 深度 | 产出形式 |
|---------|---------|------|---------|
| **小型趋势总结** | 主题累计 **20-30篇** 或 轻量比较≥3次 | 快速总结 | Markdown笔记 (1000-2000字) |
| **中型专题趋势** | 主题累计 **40-60篇** 或 中等比较≥2次 | 系统分析 | 趋势报告 (3000-5000字) |
| **阶段性全局报告** | 主题累计 **80篇+** 或 深度比较≥2次 | 全面评估 | 综合报告 (6000-10000字) |

### 2.2 维度触发规则

#### 按主线触发

```
BioMed主线:
├── drug-discovery (≥30篇) → 小型趋势总结
├── RNA-therapeutics (≥30篇) → 小型趋势总结
└── [累计≥60篇] → 中型专题: BioMed药物方向全景趋势

CompTheory主线:
├── attention-mechanisms (≥30篇) → 小型趋势总结
├── low-rank-methods (≥30篇) → 小型趋势总结
├── lora-analysis (≥30篇) → 小型趋势总结
└── [累计≥60篇] → 中型专题: 低秩方法在深度学习中的应用趋势

DeepSeek专项:
├── MLA相关 (≥15篇) → 立即小型趋势 (高优先级)
├── DeepSeekMoE (≥15篇) → 立即小型趋势
└── [累计≥40篇] → 中型专题: DeepSeek架构创新全景分析
```

#### 按作者线触发

| 触发条件 | 报告类型 | 目的 |
|---------|---------|------|
| ZSJ论文 ≥30篇 | 作者线趋势报告 | 研究演化分析 |
| ZSJ + 其他团队各≥20篇 | 跨团队趋势对比 | 方法路线格局 |
| 年度总结 | 全年趋势回顾 | 整体评估与规划 |

#### 按时间间隔触发

```
定期节奏:
├── 每季度: 该季度入库论文趋势小结
├── 每半年: 中型专题趋势报告
└── 每年: 阶段性全局趋势报告 + 研究空白分析

里程碑节奏:
├── 开题前: 全面研究空白分析
├── 投稿前: 相关方向趋势更新
└── 重大方向调整时: 重新评估趋势
```

### 2.3 推荐节奏

```
日常节奏 (每周):
├── 单篇入库 (主要工作)
├── 触发检查: 是否有主题达到比较/趋势阈值?
└── 更新趋势跟踪笔记

月度节奏:
├── 完成所有触发的轻量比较
├── 更新主题趋势跟踪表
└── 标记可能进入趋势报告的主题

季度节奏:
├── 小型趋势总结 (所有达30篇的主题)
├── 中型专题趋势报告 (达60篇的主题)
└── 初步研究空白识别

年度节奏:
├── 阶段性全局趋势报告
├── 全面研究空白分析
├── 选题建议与研究规划
└── 方法论反思与规则更新
```

### 2.4 触发决策树

```
新论文入库/比较完成
    ↓

    ↓
是否达到20篇且3个月内未做趋势总结? ──No──► 继续常规工作
    ↓ Yes
触发小型趋势总结
    ↓
是否达到40篇且6个月内未做专题报告? ──No──► 跳过
    ↓ Yes
触发中型专题趋势报告
    ↓
是否达到80篇且12个月内未做全局报告? ──No──► 跳过
    ↓ Yes
触发阶段性全局趋势报告 + 研究空白分析
```

#### BioMed 主题趋势

| 子主题 | 趋势焦点 | 关键分析维度 |
|--------|---------|-------------|
| **Drug Discovery** | 生成模型演化 | VAE→GAN→Diffusion→Flow 的迁移路径 |
| **Drug Repositioning** | 网络方法发展 | 网络传播→嵌入学习→多组学整合的演进 |
| **RNA Therapeutics** | 递送机制创新 | 脂质纳米颗粒→聚合物→外泌体的技术迭代 |
| **Multi-omics Integration** | 整合策略变化 | 早期统计方法→深度学习→Foundation Models |

#### CompTheory 主题趋势

| 子主题 | 趋势焦点 | 关键分析维度 |
|--------|---------|-------------|
| **Attention Efficiency** | 效率优化路线 | Full Attention→Sparse→Linear→Latent 的压缩路径 |
| **Low-Rank Methods** | 近似策略演化 | SVD→Random Projection→Learned Compression |
| **LoRA Adaptation** | 适配机制发展 | 简单低秩→动态rank→与其他PEFT方法融合 |

| 子主题 | 趋势焦点 | 关键分析维度 |

## 五、研究空白分析的目标

### 5.1 核心作用

1. **发现未解问题**: 识别当前方向中尚未被充分解决的问题
2. **识别薄弱环节**: 发现已有路线中的薄弱点
3. **区分真假空白**: 区分"真正空白"和"看起来冷门但无价值"
4. **支持选题决策**: 形成更有针对性的选题判断
5. **发现切入点**: 识别理论切入、方法改进、实验设计改进点

### 5.2 空白类型定义

| 类型 | 定义 | 示例 |
|------|------|------|
| **真空白** | 重要但未解决 | MLA的理论保证缺失 |
| **假空白** | 看似空白但无价值 | 已被证明不可行的问题 |
| **暂时空白** | 技术限制待突破 | 超大规模模型的训练效率 |
| **隐性空白** | 被忽视的重要问题 | 长尾样本的性能保障 |

| **无明确判断** | 要求给出具体建议 | 检查是否有可执行建议 |

### 10.2 真空白识别标准

一个真正的研究空白应满足：
- [ ] **重要性**: 解决后有实质性学术或应用价值
- [ ] **可行性**: 在当前技术条件下有可能解决
- [ ] **未被充分研究**: 不是已经被广泛研究的问题
- [ ] **不是我个人的无知**: 确认是领域共识的空白
