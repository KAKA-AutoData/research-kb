# 横向比较触发与执行规则说明

> **版本**: v1.0  
> **日期**: 2026-02-28  
> **阶段**: Phase 1 - 横向比较机制标准化  
> **性质**: 知识库从"存档"到"结构化判断"的升级规范

---

## 一、总体目标

### 1.1 为什么需要横向比较

| 问题 | 单篇分析的局限 | 横向比较的解决 |
|------|---------------|---------------|
| 知识库停留在堆积 | 只见树木不见森林 | 形成方法谱系和路线认知 |
| 难以理解方法演进 | 孤立看每篇的创新 | 识别真正推进 vs 轻微变体 |
| 难以提炼可借鉴元素 | 每篇独立，难比较优劣 | 对比中识别最佳实践 |
| 难以支持选题决策 | 缺乏宏观视角 | 识别空白和趋势 |

### 1.2 核心作用

1. **避免堆积**: 从单篇归档转向结构化理解
2. **路线认知**: 理解同一方向中的方法路线差异
3. **价值判断**: 识别真正推进 vs 轻微变体
4. **提炼借鉴**: 提炼方法设计、实验设计、理论切入的最佳实践
5. **决策支持**: 为趋势分析、研究空白、写作支持、选题提供基础
6. **判断能力**: 让知识库形成"结构化判断能力"而非只是存档能力

### 1.3 黄金标准

> **一次合格的横向比较，应该让我能够：**
> - 画出该方向的方法谱系图
> - 指出哪几篇是里程碑，哪几篇是增量改进
> - 判断哪个方法路线最适合我的问题
> - 识别该方向的共性局限和研究空白
> - 决定下一步该深入读哪类论文

## 二、触发机制

### 2.1 数量触发阈值

| 比较层级 | 触发条件 | 比较深度 | 产出形式 |
|---------|---------|---------|---------|
| **轻量比较** | 同一主题累计 **5-10篇** | 快速对比 | Markdown笔记 |
| **中等比较** | 同一主题累计 **15-20篇** | 系统分析 | 比较报告 |
| **深度比较** | 同一主题累计 **30篇+** 或阶段性节点 | 谱系构建 | 正式阶段性报告 |

### 2.2 维度触发规则

#### 维度1: 主题内触发

```
BioMed主题:
├── drug-discovery (≥5篇) → 轻量比较
├── drug-repositioning (≥5篇) → 轻量比较
├── RNA-therapeutics (≥5篇) → 轻量比较
└── [累计≥15篇] → 中型比较: BioMed药物方向方法谱系

CompTheory主题:
├── attention-mechanisms (≥5篇) → 轻量比较
├── low-rank-methods (≥5篇) → 轻量比较
├── lora-analysis (≥5篇) → 轻量比较
└── [累计≥15篇] → 中型比较: 低秩方法在NLP中的应用

DeepSeek专项:
├── MLA相关 (≥3篇) → 立即轻量比较 (高优先级)
├── DeepSeekMoE (≥3篇) → 立即轻量比较
└── [累计≥10篇] → 中型比较: DeepSeek架构创新分析
```

#### 维度2: 作者线触发

| 触发条件 | 比较类型 | 目的 |
|---------|---------|------|
| 郑双佳论文 ≥10篇 | 作者线内部比较 | 研究演化分析 |
| 郑双佳 + 其他高水平团队各≥5篇 | 跨团队比较 | 方法路线对比 |
| DeepSeek官方 + 外部研究各≥5篇 | 生态比较 | 核心vs外围评估 |

#### 维度3: 时间/路线触发

```
年度回顾:
├── 每季度末: 该季度入库论文轻量比较
├── 每年末: 全年论文中型比较 + 趋势判断

方法路线:
├── 同一问题出现3种以上不同路线 → 路线比较
├── 某路线连续3篇以上增量改进 → 路线成熟度评估
```

### 2.3 推荐节奏

```
日常节奏 (每周):
├── 单篇入库 (主要工作)
└── 触发检查: 是否有主题达到5篇阈值?

定期节奏 (每月):
├── 轻量比较: 所有达到5篇阈值的主题
└── 更新主题索引和比较清单

阶段性节奏 (每季度):
├── 中型比较: 达到15篇的主题
├── 作者线比较: 郑双佳等核心作者
└── 产出趋势判断和空白识别

里程碑节奏 (每年):
├── 深度比较: 达到30篇的核心方向
├── 全面方法谱系构建
└── 正式研究报告产出
```

### 2.4 触发决策树

```
新论文入库
    ↓
更新主题计数
    ↓
是否有主题达到5篇? ──No──► 继续单篇入库
    ↓ Yes
该主题是否已有轻量比较? ──No──► 触发轻量比较
    ↓ Yes
是否达到15篇且3个月内未做中型比较? ──No──► 跳过
    ↓ Yes
触发中型比较
    ↓
是否达到30篇且6个月内未做深度比较? ──No──► 跳过
    ↓ Yes
触发深度比较
```


```
- 优势场景与劣势场景
- 性能天花板与计算成本
- 与我问题的匹配度
```

## 四、横向比较的层级

### 4.1 三层比较体系

| 层级 | 名称 | 触发条件 | 深度 | 产出 |
|------|------|---------|------|------|
| **L1** | 轻量横向比较 | 5-10篇 | 快速对比 | 比较笔记 (500-1000字) |
| **L2** | 中等横向比较 | 15-20篇 | 系统分析 | 比较报告 (2000-4000字) |
| **L3** | 深度横向比较 | 30篇+ | 谱系构建 | 阶段性报告 (5000-8000字) |

### 4.2 轻量横向比较 (L1)

#### 适用场景
- 小批量补档后的快速总结
- 同一主题下少量论文的初步比较
- 每月例行维护

#### 目标与深度
- **目标**: 快速识别差异、代表性和值得深挖的条目
- **深度**: 表面特征对比，不做深入机制分析
- **时间**: 30-60分钟完成

#### 内容结构

```markdown
# 轻量比较: [主题] ([N]篇)

## 论文清单
| # | 论文 | 年份 | 核心方法 | 一句话评价 |
|---|------|------|---------|-----------|
| 1 | ... | ... | ... | ... |

## 快速分类
- **代表性工作**: [列出]
- **增量改进**: [列出]
- **待深入**: [列出]

## 关键差异
- 方法路线: A vs B vs C
- 性能排序: X > Y > Z
- 实验强弱: [简要评价]

## 最值得深挖
1. [论文名] - [原因]
2. [论文名] - [原因]

## 下一步建议
- [ ] 补充哪类论文
- [ ] 哪篇需要完整深度分析
```

### 4.3 中等横向比较 (L2)

#### 适用场景
- 某主题已有一定密度 (15-20篇)
- 某作者线已有若干代表论文
- 季度性回顾

#### 目标与深度
- **目标**: 比较方法路线、实验设计和局限性，形成判断
- **深度**: 理解各方法的核心逻辑和适用场景
- **时间**: 2-4小时完成

#### 内容结构 (8章)

```markdown
1. comparison-scope      # 比较范围与选择理由
2. problem-setting       # 问题设定对比
3. method-lineage        # 方法路线梳理
4. key-differences       # 核心差异分析
5. experiment-comparison # 实验设计对比
6. result-assessment     # 结果质量评估
7. limitation-patterns   # 共性局限总结
8. follow-up-suggestions # 后续建议
```

### 4.4 深度横向比较 (L3)

#### 适用场景
- 某主题或专项线已经积累到足够规模 (30篇+)
- 需要产出正式的阶段性比较报告
- 年度总结或开题前论证

#### 目标与深度
- **目标**: 形成方法谱系、趋势判断、研究空白与选题建议
- **深度**: 构建完整认知框架，可服务于综述式写作
- **时间**: 1-2周完成

#### 内容结构 (9章 + 附录)

```markdown
1. executive-summary     # 执行摘要
2. scope-and-methodology # 范围与方法
3. historical-context    # 历史背景
4. method-taxonomy       # 方法分类学
5. detailed-comparison   # 详细比较
6. trend-analysis        # 趋势分析
7. gap-identification    # 空白识别
8. recommendations       # 建议
9. conclusion            # 结论

附录:
- 完整论文清单
- 数据表格
- 方法谱系图
```
